================================================================================
HOW IT WORKS
================================================================================

Search Workflow:
  1. User enters book name → Submit
  2. Request added to queue
  3. Bot connects to IRC (random nickname)
  4. Bot sends "@Search <book name>"
  5. SearchBot sends zip file via DCC
  6. Bot downloads, extracts, and parses results
  7. Results filtered for .epub/.mobi/.pdf
  8. Results grouped by bot and sent to client
  
Download Workflow:
  1. User selects books → Download
  2. Commands added to queue
  3. Bot sends each "!BotName <file>" command
  4. Bot receives files via DCC
  5. Files moved to library directory
  6. Metadata extracted and saved to database
  7. Client notified of completion

================================================================================
TROUBLESHOOTING
================================================================================

If bot won't connect:
  - Check IRC_PORT is 6660 (not 6667)
  - Verify firewall allows outgoing connections
  - Check IRC server is accessible

If search times out:
  - Increase REQUEST_TIMEOUT in config.py
  - Verify SearchBot is active in #ebooks channel
  - Check search query format (no special chars at start)

If downloads fail:
  - Check directory permissions for downloads/
  - Ensure enough disk space
  - Verify IRC connection status in navbar

See WEB_INTERFACE.md for complete troubleshooting guide.
